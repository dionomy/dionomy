# MVP 구현 로드맵

## 진행 방식

각 단계는 다음 루프로 진행한다.

```txt
계획
구현
검증
commit-plan-and-split 기준 커밋 분할
커밋
```

커밋은 최소 기능 단위로 나눈다.

## 단계 0. 저장소 기준선

목표:

- 루트 메타 저장소 기준 확정
- 프론트엔드와 백엔드 독립 저장소 준비
- 문서, 기술 스택, MVP 범위 기준선 커밋

산출물:

- 루트 문서
- `frontend` 저장소
- `backend` 저장소
- submodule 포인터

## 단계 1. 앱 골격

목표:

- React/Vite 프론트엔드 기본 앱 구성
- Kotlin Spring Boot 백엔드 기본 앱 구성
- feature-based + DDD 구조 생성
- Figma `Web / Dashboard`, `Web / Schedule` 기준 화면 골격 연결

산출물:

- 프론트 라우팅
- 공통 레이아웃
- 주요 feature 폴더
- 백엔드 도메인 모듈 폴더
- 헬스 체크 API

## 단계 2. 인증과 테넌트

목표:

- 카카오 로그인 연동 전 내부 인증 흐름 준비
- 사용자 역할 모델 구성
- tenant scope 정책 적용

산출물:

- 사용자/역할 도메인
- 테넌트 도메인
- 임시 로그인 API
- 권한별 화면 진입

## 단계 3. 학원 설정과 화이트라벨

목표:

- 학원 기본 정보
- 로고/메인 컬러
- 수강권/보강 정책 설정

산출물:

- 학원 설정 API
- 원장 설정 화면
- 관리자 온보딩 화면

## 단계 4. 일정과 수업

목표:

- 그룹/1:1 수업 등록
- 반복 일정
- 수업 이동/취소
- 강사/장소 충돌 검사
- 수강생 배정
- Figma 시간표 동시간대 표시 규칙 반영
- Figma 부분 겹침 레이아웃 알고리즘 반영

산출물:

- 수업 일정 도메인
- 캘린더 API
- 원장 통합일정 UI

## 단계 5. 수강생과 수강권

목표:

- 수강생 등록/조회
- 수강권 상품 등록
- 수강권 발급
- 회차 차감/복구 로그
- 자동 만료

산출물:

- 수강생 도메인
- 수강권 도메인
- 수강생 상세 UI

## 단계 6. 출석, 결석, 보강

목표:

- 강사 출석 체크
- 수강생 결석 신청
- 강사 승인
- 다른 세션 이동 또는 보강 처리

산출물:

- 출석 도메인
- 결석 신청 도메인
- 보강 도메인
- 강사 모드 UI
- 수강생 내 일정 UI

## 단계 7. 클래스노트와 공지

목표:

- 수업 단위 클래스노트 작성/열람
- 공지 작성/발송 대상 지정
- 공지 목록/상세

산출물:

- 클래스노트 도메인
- 공지 도메인
- 수강생 읽기 화면

## 단계 8. CRM 위험 신호

목표:

- 룰 기반 위험 신호 계산
- 위험 수강생 리스트
- 케어 기록
- 대시보드 To-do 반영

산출물:

- CRM 도메인
- 매일 갱신 배치
- 원장 CRM 화면

## 단계 9. 회사 웹과 관리자 세팅 웹

목표:

- 회사 랜딩
- 데모 신청
- CS 문의
- 학원 목록/온보딩
- 빌드 상태 모니터 기본 UI

산출물:

- 회사 웹 화면
- Dionomy 관리자 화면
- CS 티켓 기본 모델

## 단계 10. 통합 검증

목표:

- MVP 핵심 플로우 검증
- 테스트 보강
- 배포 준비

검증 플로우:

- 원장이 수업 생성
- 수강생 배정
- 강사가 출석 체크
- 수강생이 결석 신청
- 강사가 승인
- 수강권 회차가 차감/복구
- 클래스노트가 수강생에게 노출
- 위험 신호가 대시보드에 표시
